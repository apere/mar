On this page, I'll describe the the workflow of updating the website mareshiadonald.com. I will attempt to be as detailed as possible by listing the software that you'll need; how to download the current site's content; the files that you may want to edit; templates for editing particular sections; and how to upload to the site.

Software

FileZilla
This is the software that you will be using to connect to the server located at mareshiadonald.com. Filezilla is an FTP (File Transfer Protocol) application. It allows you to view and update files on an FTP server in the same way you would view and update files on your own computer.
Brackets
This is my personal favorite text editor for coding. This application will let you open up any of your website files and edit them. It works just like a regular text editor but does some smart color coding for editing code. If you don't like brackets, you may want to try Sublime Text.

Setting up Filezilla

Filezilla is very easy to get up and running. Start by downloading and then installing the application.

  1. After you've installed filezilla, open it for the first time and you should see this screen
  2. Click the Site Maganger link
  3. Click New Site and enter your information like below. You should already know your password and username. Click Connect and that will save your settings and connect to the server.
    Host
    ftp.mareshiadonald.com
    Protocol
    FTP - File Transfer Protocol
    Encryption
    Use plain FTP (insecure)
    Logon Type
    Normal
    User
    Your username (most likely the one in the image)
    Password
    Your pasword
  4. If you get a message prompting you to trust the servers certificate, click ok .
  5. Select every file and save everything in it by dragging it into a folder on your computer (if you want, you can download a copy of the website from the last time adam edited here). This folder contains entire contents of your website. This is where you'll be working from.

  • /public/images/

    All images belong in this folder. If you want to swap out an image, I suggest naming it the same as the image you're replacing and swap the files.

  • index.html

    Every webpage will have a filename of "index.html". This one, the one in the main folder, is the file for MareshiaDonald.com/

    To get started, find the block of code with the class inner-wrapper.

    <div class="inner-wrapper">
  <h1>Mareshia Donald, PhD</h1>
  <h5>Scientist, STEM Educator and Outreach Professional with a commitment to social justice and diversifying the STEM pipeline.</h5>
</div>

    Inside of inner-wrapper, the two places that you'll want to edit are:

    1. To edit the title use 
      <h1> [ insert title ] </h1>
      e.g.

      Mareshia Donald, PhD

    2. To edit the phrase use 
      <h5> [ insert phrase ] </h5>
      e.g.
      Scientist, STEM Educator and Outreach Professional with a commitment to social justice and diversifying the STEM pipeline.

Code Reference

This section will briefly describe every HTML element that was highlighted above.

Structure of an HTML element

An HTML element is a chunk of text that describes how the chunk of text inside of it will be displayed on a webpage. The basic structure of every HTML element is the same:

<div class = "description">   …    </div>
  1. The element itself is denoted with greater than and less than symbols 
    <div class = "description">   …    </div>

    The first pair denotes the opening of element and the second pair (which contains a backslash) denotes the ending of the element. Everything between the two pairs is the actual content that will be displayed on the page.

  2. The opening and closing of the element always contains the type of element 
    <div class = "description">   …    </div>

    In this case, the type of element being used is a div

  3. Sometimes, an element will have a class. These classes are used to style elements correctly. The classes is denoted byclass = "…" and is always located in the opening of the element. 
    <div class = "description";>   …    </div>

    In this case, our div has a class of description

  4. Some elements have special attributes, one of those is a link which contains an href. This is what defines the url for the link. 
    <a href= " https://www.google.com " > link to google </a>

    In this case, we have a link that points to google.com with the text link to google.

  5. Finally, the actual contents of the element (or the text that will be displayed on your webpage) is always located between the opening and closing of an element. 
    <div class = "description">   This is my description text.   </div>

    In this case, our div will display the text This is my description text.

Classes you may use

  1. inner-wrapper — On the homepage, this class is used to define the area where the main title and subtitle are held.
  2. date — This class is used when the element contains a date (surprise). This is used on the Awards & Honors page.
  3. link — This class is used when a definition description contains a link. This is used on the Awards & Honors page.
  4. address — This class is used when a definition description contains an address or location. this is used on the Education & Experience page and the Service page.
  5. pdTitle — This class is used when a definition term contains a title on the Education & Experience page and the Service page.
  6. description — This class is used when a definition description contains a description paragraph (or more) on the Education & Experience page and the Service page.
  7. service-item — This class is used when a definition term contains the title of a service item on the Service page.

HTML elements you may use

  1. div — This element simply defines a block of content that should exist on its own row.
  2. a — This is a link element. Place the url for where you want the link to point to in the href attribute.
  3. h1 — This is the largest header element. It is only used on the homepage
  4. h2 — This is the second largest header element. It is used to denote the beginning of a section.
  5. h5 — This is the smallest header element. It is only used on the homepage to denote the subtitle.
  6. p — This is a paragraph element. It is used to define a paragraph of text.
  7. strong — This is used to denote text that should be bolded.
  8. em — This is an emphasize element. This is used to make any text light gray.
  9. ul — This is an unordered list. This list should contain a series of lielements whose order does not matter. (e.g. a bulleted list).
  10. ol — This is an ordered list. This list should contain a series of lielements whose order does matter. This adds a number to the left of each element (e.g. this list).
  11. li — This is a list item. This should always be inside of anolor ul.
  12. dl — This is a definition list. This is a special kind of list that denotes a series of terms and their corresponding "defenitions". A definition list should contain a series of dt and ddelements.
  13. dt — This is a definition term. This is used to denote the 'term' for the next set of dd elements.
  14. dd — This is a definition description. This contains any elements that are used to describe the preceeding dt. You can have as many definition descriptions per term as you would like.
